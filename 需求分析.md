# 部门协作流程 · 需求分析

> 配套文档：《技术方案》(`技术方案.md` / `.html`)、UI 设计稿 (`demo/增量UI设计图.html`)。
> 范围：在现有「同辉智能项目管理系统」基础上，增量接入「多部门下单协作 → 下游推送 → 物流发货 → 报表 / 审批 / 通知」。

---

## 一、背景与目标
现有系统已能管理项目（项目目录/项目详单）、维护项目内 4 张进度数据表、按角色做字段级权限。但**跨部门的任务派发与协同**目前靠线下沟通：设计、电工、生产各自为政，完成后靠口头/微信通知下游，进度与逾期无系统化跟踪。

本次目标：把「下单 → 接单 → 完成 → 通知下游 → 物流发货」整条链搬到系统里，并提供完成效率、月度报表、导出管控等管理视图。

## 二、当前系统业务现状（与本次相关）
| 现状 | 说明 |
|---|---|
| 项目目录（一览） | 14 列表格，含状态、签订/交货日期、制图开始/结束、电工、货期、剩余制作时间等；状态可切，剩余时间按今天实时算（已完成则冻结） |
| 项目详单 | 卡片视图，点进项目详情 |
| 项目详情 | 每个项目自带 4 张数据表：钣金装配 / 标准件清单 / 外协外购 / 原料下料单 |
| 角色 | 固定 8 个：超级管理员、管理层、设计师、生产文员、仓库员、标准件采购员、外协机加工采购员、人事行政 |
| 权限 | 项目级（成员 edit/view）+ 字段级（按角色 can_view/can_edit） |
| 导出 | 项目一览可导出 Excel（当前无审批） |

## 三、需求范围与总流程
```
管理层/主管 在项目目录「项目编号」上下单
   → 指派部门(设计/电工/生产) + 负责人 + 要求 + 资料
   → 负责人接单填「开始/预计完成」时间并开始
   → 完成（设计部先校验四表 / 选通知人 / 上传产物）
   → 产物推送下游（采购部 / 物流发货部）+ 企业微信通知
   → 逾期则推送对应部门主管
物流发货部：新建项目即生成发货待办，汇总各部门资料 → 维护收货信息 → 上传发货单 → 已发货
管理视图：月度工作报表、导出权限申请/审批
```

## 四、功能需求

### 4.1 角色（改动）
- 新增：电工、装配组1/2/3、物流发货部、采购部（合并原两个采购员）、设计/电工/生产部主管。
- 「消息推送人」：标记后接收对应部门逾期等推送。
- 下单权限：管理层→设计/电工部；生产部主管→生产部。

### 4.2 下单与执行
| 编号 | 需求 |
|---|---|
| F1 | 项目目录「项目编号」上浮出「＋下单」（仅有权限角色可见），弹窗选部门/负责人/要求/资料(PDF/Word/Excel) |
| F2 | 负责人在本部门工作台收到待办 |
| F3 | 接单填「开始/预计完成」并开始；设计部「制图开始」回传项目目录 |
| F4 | 完成可逆（已完成 ↔ 进行中可来回切） |
| F5 | 完成效率 = 实际用时 ÷ 预计用时（≤100% 绿、>100% 红） |
| F6 | 项目编号链接到项目详情的具体表格 |

### 4.3 完成校验 / 完成后
| 编号 | 需求 |
|---|---|
| F7 | 设计部完成前校验该项目「四个 Excel 表」齐全，缺则禁止完成 |
| F8 | 完成后通知具体人员（设计部自选 / 电工部→采购部 / 生产部→物流部）→ 企业微信 |
| F9 | 各部门产物上传与推送：设计部**完成**时传 说明书(Word)+铭牌(CAD)→物流；电工部**接单后**传 采购清单(Excel)→采购，**完成时必传**电路图(PDF)→物流（未传不可完成）；生产部无产物 |
| F10 | 逾期 → 推送对应部门主管 |

### 4.4 下游与物流
| 编号 | 需求 |
|---|---|
| F11 | 采购部接收电工部推送的采购清单（收件箱） |
| F12 | 物流发货部：新建项目自动建发货待办；各部门资料按 设计/电工/生产 三栏汇总；维护收货信息；上传发货单→已发货 |

### 4.5 报表 / 审批 / 通知
| 编号 | 需求 |
|---|---|
| F13 | 月度工作报表：按部门 + 个人统计任务量/按时率/平均效率，含对比图与逾期清单 |
| F14 | 导出审批：成员申请 → 管理层审批 → 通过后导出 |
| F15 | 企业微信通知：完成、逾期、审批结果，落消息中心 + 推企微 |

### 4.6 非功能需求
- 现有页面（项目目录/详单/进度表/权限）UI 不变，仅叠加增量。
- 附件类型白名单、大小限制、鉴权下载。
- 企微 token 缓存、失败降级（不影响主流程，站内兜底）。
- 逾期定时任务多实例幂等。

## 五、需求澄清问题（结合现有系统，待业务方确认）

> 以下问题直接影响建模与交互，建议逐条确认；标注了与现有系统的关联点。

### A. 角色与权限
- **A1** 现有「生产文员 / 仓库员 / 人事行政」在新流程中的定位？是否参与下单/接单，还是仅沿用原字段权限？
- **A2** 采购部合并后，原「标准件采购员 / 外协机加工采购员」的**存量用户与已配字段权限**如何处理（自动并入 purchasing？）？
- **A3** 各部门页面的可见范围：仅本部门成员 + 管理层，还是所有登录用户都能看？
- **A4** 管理层是否可越权给**生产部**下单（当前定生产部主管下单）？

### B. 下单与执行
- **B1** 同一项目、同一部门能否**重复下单**（返工再派）？重复是新建任务还是覆盖原任务？
- **B2** 可下单的项目范围：所有项目，还是仅「进行中」？已完成/已归档能否下单？
- **B3** 设计部「制图开始」回传项目目录时，若该列**已有手填值**，是否覆盖？（现状：该列是可手填的 meta 列）
- **B4** 一个任务能否指派**多人**（如装配组协作），还是只能一个负责人？
- **B5** 开始时间、预计完成时间在进行中**能否修改**？

### C. 时间与效率口径
- **C1** 完成效率 / 逾期按**自然日**还是**工作日**（扣周末节假日）？（现状：派生列用自然日）
- **C2** 预计完成**当天**完成，算按时还是逾期？（边界 done == due）
- **C3** 预计用时为 0（开始=预计完成同一天）时，效率如何计算？
- **C4** 月度报表按任务的**完成时间**还是**下单时间**计入当月？
- **C5**「按时率」= 按时数 ÷ 已完成数，还是 ÷ 全部任务数？

### D. 完成动作与下游
- **D1**「四表已上传」如何判定？4 张表各有 ≥1 行数据算，还是必须**导入过 Excel**？手工加行算不算？（现状：每项目自带 4 张空表）
- **D2** 四表校验是否**只卡设计部**完成，电工/生产完成不校验？
- **D3** 设计部「自选通知人」范围：全公司任意人，还是限定相关角色？能否**多选/抄送**？
- **D4**〔已明确〕电工部：采购清单(Excel)**接单后**上传→采购；电路图(PDF)**完成时必传**→物流。延伸：设计部 说明书/铭牌 是否也设为**必传**才能完成？
- **D5** 物流**发货前置条件**：是否要求设计/电工/生产都完成才可发货，还是物流可随时发？

### E. 物流与收货
- **E1** 一个项目是否只发一次货（一条发货待办）？是否存在**分批发货**（多张发货单）？
- **E2** 收货信息来源：物流**手工录入**，还是从客户/合同带出？一个项目是否可能**多个收货地址**？
- **E3** 生产部不产出图纸 → 物流三栏的「生产部」恒为空，是否需要生产部完成时上传「完工/质检单」补到该栏？

### F. 企业微信与通知
- **F1** 系统用户与企微成员的**映射方式**（手机号 / 工号 / 企微 userid）？无企微账号者如何通知（仅站内？）
- **F2** 通知是即时单条还是可**合并**（每日汇总）？逾期是否**每天重复提醒**直到处理？
- **F3** 站内消息是否需要「已读/未读 + 待办中心」，还是仅铃铛提示？

### G. 导出与审批
- **G1** 哪些导出需要审批？现有「项目一览导出 Excel」是否也纳入，还是仅新增报表/清单导出？
- **G2** 审批人固定为管理层，还是按数据范围找对应负责人？审批**超时**怎么处理？
- **G3** 导出格式：仅 Excel，还是也要 PDF？

### H. 范围与上线
- **H1** 新流程对**存量项目**（已有数十个进行中项目）是否追溯，还是只对新建项目生效？
- **H2** 上线方式：灰度（部分部门先用）还是全量切换？

## 六、已确认决策
1. **部门主管**：新建 `design_lead / electric_lead / produce_lead` 三角色，逾期推送给对应部门主管。
2. **企业微信**：自建应用尚未具备 → 先做站内通知，企微接口预留，凭证到位再接，不阻塞前序。
3. **装配组**：`assembly_1 / assembly_2 / assembly_3` 三个独立角色。
4. **导出审批**：一次性放行（每次导出都申请审批一次）。
5. **完成效率口径**：实际用时 ÷ 预计用时（数值越低越好）。
6. **各部门产物**：设计部完成传 说明书(Word)+铭牌(CAD)→物流；电工部**接单后**传 采购清单(Excel)→采购、**完成必传**电路图(PDF)→物流；生产部无产物。
7. **生产部下单**：生产部主管在项目目录下单、选装配组1/2/3，**下单无需上传资料**（设计/电工下单仍可附资料）。
