# 仓库管理功能 · 单独设计方案

> 配套：《需求分析》《技术方案》、UI 设计稿 `demo/增量UI设计图.html`、仓库原型 `demo/仓库管理-原型.html`。

## 一、现状（来自样本 `搅拌桨库存表.et`）

当前每类物料一个 Excel 工作簿（如「搅拌桨库存表」），**按月一个工作表**（1月/2月/3月/5月/6月…）。每张表结构：

| 材料名称 | 型号 | 上月库存数 | 日期 | 1 | 2 | … | 31 | 入/出库总数 | 库存数量 |
|---|---|---|---|---|---|---|---|---|---|
| 304搅拌浆 | 5L | 25 | **入** | | | | 2 | 2 | 25 |
| | | | **出** | | | 2 | | 2 | |

- 每个物料占**两行**：一行「入」、一行「出」；1–31 列按当天填入/出数量。
- `入/出库总数` = 当行求和；`库存数量` = 上月库存数 + 本月入 − 本月出（写在「入」行）。
- 每月**手工把上月期末抄成本月「上月库存数」**。

### 暴露出来的问题（正是上系统的理由）
| 问题 | 样本证据 | 系统化后如何解决 |
|---|---|---|
| 物料命名不统一 | 「304搅拌**桨**」与「304搅拌**浆**」混用（桨/浆错字） | 物料主数据唯一编码，名称受控、不再自由手打 |
| 型号漂移导致同物料被当成两个 | 同一行 3 月起 `15L` → `15L/20L` | 物料身份由**编码**固定，型号是属性，变更留痕 |
| 重复/版本行 | 出现 `304搅拌浆（老）`/`（新）`、两条 `316搅拌桨 5L` | 主数据合并 + 版本/启停管理 |
| 期初靠手抄、易错 | 每月手填「上月库存数」 | 月度期初=上月期末，自动结转 |
| 库存非实时、易算错 | 库存数量手工算在「入」行 | 实时库存 = 期初 + Σ入 − Σ出，自动 |
| 月份断档 | 缺 4 月 | 流水按日期连续，无"缺表"概念 |
| 多簿分散、口径不一 | 一类一簿 | 一套主数据 + 分类维度统一管理 |
| 与项目脱节 | Excel 无法关联项目领用 | 出库可挂项目编号，可追溯 |

## 二、设计原则
> Excel 是**工厂现状、不是目标**。按行业通行的轻量 WMS 思路重做，而非照搬那张又宽又笨的 31 列日表。

1. **纯增量**：新增「仓库组」一级菜单 + 全新数据表 + 新 router，不动 `项目目录/项目详单/进度表/权限` 等任何现有页面与表。
2. **单据化、库存自动结算**：出入库以**单据**承载（自动单号、留痕、可冲红），`物料主数据` + `出入库流水` →（自动算）`实时库存`；月初自动结转期初。
3. **专业报表替代日表**：用行业标准的**收发存汇总**（期初 + 入 − 出 = 期末）替代 31 列月度日表，清爽可对账；**保留**一键导出 Excel/PDF 的能力（含可选"老版式"用于过渡对账），但系统内体验是现代化的。
4. **好用优先**：库存看板（KPI + 安全库存预警 + 建议补货）、库位、搜索筛选、出库挂项目领料追溯。
5. **对齐现有技术栈**：SQLAlchemy 2.0 async + `PortableJSON`、`data_migration.run_all` 幂等建表/导入、`seed.py` 同步角色、复用消息中心与导出审批。

## 四、数据模型（新增表，零侵入）

```python
# models.py 追加（命名空间 wh_ 前缀，互不影响现有表）

class WhMaterial(Base):              # 物料主数据
    __tablename__ = "wh_materials"
    id:           Mapped[int]  = mapped_column(primary_key=True)
    code:         Mapped[str]  = mapped_column(String(64), unique=True, index=True)  # 物料编码，稳定身份
    name:         Mapped[str]  = mapped_column(String(128), index=True)              # 材料名称（受控）
    spec:         Mapped[str]  = mapped_column(String(64), default="")               # 型号/规格
    category:     Mapped[str]  = mapped_column(String(64), index=True, default="")   # 分类，如「搅拌桨」
    unit:         Mapped[str]  = mapped_column(String(16), default="个")             # 单位
    location:     Mapped[str]  = mapped_column(String(32), default="")               # 库位（如 A-01；单仓可留空）
    safety_stock: Mapped[int]  = mapped_column(default=0)                            # 安全库存阈值
    init_stock:   Mapped[int]  = mapped_column(default=0)                            # 建档期初
    status:       Mapped[str]  = mapped_column(String(16), default="启用")           # 启用/停用
    remark:       Mapped[Optional[str]] = mapped_column(String(255))
    created_at:   Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    __table_args__ = (UniqueConstraint("name", "spec", name="uq_wh_material_name_spec"),)

class WhTxn(Base):                   # 出入库流水（唯一事实来源）
    __tablename__ = "wh_txns"
    id:          Mapped[int]  = mapped_column(primary_key=True)
    material_id: Mapped[int]  = mapped_column(ForeignKey("wh_materials.id"), index=True)
    biz_date:    Mapped[str]  = mapped_column(String(10), index=True)   # YYYY-MM-DD（复用全局日期规范）
    direction:   Mapped[str]  = mapped_column(String(4))                # in / out
    qty:         Mapped[int]  = mapped_column()                         # 正整数
    source:      Mapped[str]  = mapped_column(String(32), default="")   # 采购入库/项目领用/发货出库/盘点调整/期初
    project_id:  Mapped[Optional[int]] = mapped_column(ForeignKey("projects.id"))  # 关联项目（可空）
    ref_no:      Mapped[Optional[str]] = mapped_column(String(64))      # 单据号
    operator_id: Mapped[int]  = mapped_column(ForeignKey("users.id"))
    remark:      Mapped[Optional[str]] = mapped_column(String(255))
    created_at:  Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())

class WhMonthOpen(Base):             # 月度期初快照（替代手抄「上月库存数」）
    __tablename__ = "wh_month_opens"
    id:          Mapped[int] = mapped_column(primary_key=True)
    material_id: Mapped[int] = mapped_column(ForeignKey("wh_materials.id"), index=True)
    ym:          Mapped[str] = mapped_column(String(7), index=True)     # YYYY-MM
    open_qty:    Mapped[int] = mapped_column(default=0)
    __table_args__ = (UniqueConstraint("material_id", "ym", name="uq_wh_open_mat_ym"),)
```

**实时库存**不单独存字段，**按需计算**（口径唯一、不会算错）：
```
当前库存(m) = init_stock + Σ(in.qty) − Σ(out.qty)         # 全量
某月期末(m, ym) = open_qty(m, ym) + Σ当月in − Σ当月out      # 月度
```
> 物料多、流水大时再加 `wh_stock` 缓存表（运行余额），P1 先按需算即可。

## 五、功能与页面（「仓库组」菜单下）

```
仓库组
 ├── 库存总览        ← 看板：KPI + 安全库存预警/建议补货 + 实时库存表（设计师只读到这）
 ├── 出入库登记      ← 一个入口、入/出切换；自动单号、库位、出库挂项目领料；提交即结算
 ├── 收发存汇总      ← 期初 + 入 − 出 = 期末（按月/累计），可导出 Excel/PDF 对账（替代 31 列日表）
 ├── 出入库流水      ← 按物料/日期/类型/项目过滤，审计留痕，冲红纠错，导出
 ├── 物料主数据      ← 新增/编辑/停用、合并重复、修正错字、维护安全库存与库位
 └── 库存盘点        ← 实盘录入→差异（盘盈/盘亏）→生成调整单据，对平账面（P2/P3）

（同菜单下另有「发货清单上传」——独立子功能，见七.2）
```

| 页面 | 关键交互 |
|---|---|
| 库存总览 | KPI（物料数/需补货数/本月入/本月出）+ 预警面板（当前·安全·建议补货量）+ 库存表（编码/名称/规格/库位/库存/状态：正常·预警·缺货）；设计师/管理层只读无操作按钮 |
| 出入库登记 | 入/出切换（采购入库·退料·盘盈 / 项目领料·发货·报废·盘亏）；物料可搜、自动单号、库位带出、出库挂项目编号；出库超库存拦截；提交即结算库存 |
| 收发存汇总 | 选区间（月/累计）→ 每物料 期初/入/出/期末 + 合计行；导出走审批（口径 G1/G3：Excel+PDF）；期末恒等于实时库存 |
| 出入库流水 | 单据明细 + 过滤；任何一笔可追溯到经办人/项目/单据；冲红生成反向单据，不物理删除 |
| 物料主数据 | 唯一编码；name+spec 唯一约束防重复；数据质量标记（错字/型号漂移/老新）；「合并」把误建同物料流水迁移到主物料；Excel 历史导入 |

> 后续可扩展（原型已标注、非首期）：批次/序列号、条码扫码、ABC 分类、呆滞料预警、多仓库/多库位、库存金额核算。

## 六、与现有系统的集成点

1. **设计部只读查看库存**（需求明确）：在设计部页面/库存总览提供只读视图，复用同一查询接口，按角色隐藏所有写操作。
2. **出库关联项目**：出库登记可选「关联项目编号」（即项目目录编号），实现"哪个项目领了什么料"的追溯，为后续成本/发货核对铺路。
3. **低库存预警**：库存 < 安全库存 → 站内消息中心 + （企微就绪后）推仓库主管；复用确认口径 F2/F3（即时单条、铃铛+列表）。
4. **导出审批**：月度明细表/流水导出统一走「导出权限申请→管理层审批」（口径 G1/G2/G3：全部导出需审批、管理层审批、Excel+PDF）。
5. **企业微信**：预警/异常走既有通知通道，凭证到位即接，不阻塞。

## 八、Excel 历史数据迁移（一次性）

提供导入工具（脚本或后台页），解析现有 `*库存表.et/.xlsx`：

1. **建主数据**：按 `名称+型号` 去重；**规范化**（统一「桨/浆」、合并 `15L`/`15L/20L`、处理 `（老）/（新）`），人工确认合并表后入库，生成编码。
2. **建期初**：取每月「上月库存数」/首月值写入 `wh_month_opens` 或 `init_stock`。
3. **重建流水**：把 1–31 天的「入/出」非空单元格还原成 `WhTxn`（biz_date=该月该日，direction=入/出，qty=单元格值，source="期初导入"）。
4. **校验**：导入后按公式重算每月期末，与 Excel「库存数量」比对，差异列出人工核对。

> 口径 H1：存量纳入。建议**先导期初+主数据全量切换**，历史明细按需补导（不阻塞上线）。

## 十二、待确认（仓库专属）

1. **物料编码规则**：系统自动生成（如 `分类前缀+流水`）还是沿用现有编码/手填？
2. **分类体系**：除「搅拌桨」外还有哪些大类？是否需要多级分类？
3. **多仓库/库位**：是否只有一个仓库？要不要库位（货架）维度？（默认：单仓，不分库位）
4. **单位**：是否统一为「个/台/套」？同物料是否存在多单位换算？
5. **出库是否必须挂项目**：领用一定关联项目编号，还是允许非项目出库（如样品/损耗）？
6. **安全库存**：每个物料一个阈值，由谁维护？是否需要按月动态？
7. **历史导入深度**：只导「期初+主数据」即可上线，还是必须把每月每日明细全部还原？
8. **盘点**：多久盘一次？盘点差异是直接调账还是需审批？
9. **型号合并**：`15L` 与 `15L/20L`、`（老）/（新）` 是否合并为同一物料（型号取并集/拆分为两物料）？
```
