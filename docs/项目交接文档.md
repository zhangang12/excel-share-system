## 一、系统是什么

同辉项目管理 ERP：制造业项目全流程内部系统，覆盖 **销售→设计→电工→生产→采购→仓库→物流→财务→售后→人事** 十个业务域 + 管理层视角。核心是「项目」：一个项目从销售下单开始，依次流过各部门任务单，采购/仓库/物流保障物料与交付，财务管回款，全程消息驱动。

- 角色体系：`admin`（系统管理员）、`manager`（管理层）、各部门员工/主管（如 `buyer`/`buyer_lead`、`sales`/`sales_lead` 等，见 `backend/app/menus.py` 的 `ROLE_MENUS`）
- 菜单与权限的唯一权威：`backend/app/menus.py`；前端侧边栏按 `GET /api/auth/menus` 渲染

## 二、技术架构与部署

- **后端** FastAPI + async SQLAlchemy（`backend/app/`），生产 PostgreSQL，开发/测试 SQLite
- **前端** Vue 3 + TS + Element Plus + vxe-table（`frontend/src/`）
- **服务器** `root@8.141.123.141:/opt/pms/excel-share-system-main`，docker compose 一套容器：`pms2_backend`（uvicorn --workers 4）/ `pms2_frontend` / `pms2_nginx` / `pms2_postgres`
- **发版唯一路径**：`bash ops/release.sh --push`（本地 push main → 服务器 `upgrade.sh`：备份→拉码→重建→健康检查→失败自动回滚）。部署配置在 `.deploy.local`（不入库）；构建在服务器做
- 运维脚本（备份/回滚/日志/重置密码等）在 `ops/`，说明见 `ops/README.md`

## 三、核心机制（改代码前必读）

1. **日期口径**：日期字段多为 ISO 字符串，直接字典序比较；「今天」一律走 `overdue.py` 的 `_CN_TZ`（UTC+8），不要用 UTC 当天，否则日界错位导致重复推送
2. **消息幂等**：所有周期提醒用 `biz_type + biz_id + 当日` 在 messages 表去重；多 worker 部署靠 flock 保证单实例扫描（`overdue.py`）
3. **现有 6 个扫描器**（`overdue.py`，12h 一轮）：任务逾期、尾款到期、逾期尾款周催办、人事到期、管理层待办、采购到期未到货
4. **菜单权限**：后端 `menus.py` 下发 key 列表；前端 `MainLayout.vue` 的 `ADMIN_EXTRA` 把部分 key 归入「管理」分组；前端 auth store 的 `isAdmin` 包含 admin 和 manager
5. **建表/迁移**：新表靠启动 `create_all` 自动建；存量表加列必须在 `data_migration.py` 加幂等迁移
6. **企微通道**：`notify.py` 已打通企微应用消息外发（用户绑定 wxid + `settings.wecom_*` 配置），所有提醒都走这里

## 四、开发与测试

```bash
cd backend && .venv/bin/python -m uvicorn app.main:app --reload   # 后端
cd frontend && npm run dev                                        # 前端
```

- 种子账号：`admin / admin123`、`manager / manager123`
- 后端测试：`backend/tests/` 下 40 个**独立 asyncio 脚本**（临时库、自带数据），跑法 `cd backend && .venv/bin/python tests/xxx.py`。**不要用 pytest**
- 前端检查：`node_modules/.bin/vue-tsc -b`；`npm run build` = 类型检查 + 打包
- ⚠️ **16 个测试是历史欠账，在干净 HEAD 上就失败**（清单见 `AGENTS.md`「已知坑」）。改代码后只需保证：你碰的测试全绿 + 没把原本绿的搞红。判断是否你引入的：`git worktree add /tmp/es-base HEAD` 对比基线
