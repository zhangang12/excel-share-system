企业微信推送 · 业务逻辑确认表

用途：请业务确认每条推送「触发时机 / 接收人 / 内容」是否合理；需要调整的在最右列写「保留 / 修改 / 删除 + 意见」。版本日期：2026-06-28
推送机制（先看这段）：系统里所有「站内消息」会同步推送到企业微信——前提是该接收人已在「企微绑定」页绑定了企微账号（userid）。没绑定的人只收站内消息，不影响业务。
下表「推送内容」里 {编号} {名称} {金额} 等是占位符，实际会替换成真实数据。「接收人」分两种：具体某人（如该项目销售员/负责人本人）、某角色全体（该角色的所有在职人员都会收到）。
