#要点对应到本设计业务确认
A供应商按「账期」分类：有账期（如月结3个月）/ 无账期 / 现金。账期要体现在系统里——未付的显示为「应付/欠款」，到期才算应付。供应商档案加「结算方式 + 账期天数」；对账/报表按账期算应付与到期。见 4.1#1、4.4、4.6。
B报表口径：数据录完→老板看总报表；① 按采购员（约3个采购各一张）② 按项目一搜就知道该项目花了多少钱 ③ 每项目含 合计 / 已开票 / 待开票 / 已付款 / 欠款。汇总报表加「按采购员」「按项目搜索」两个视角；列含上述5项金额。见 4.6。
C对账闭环：一张对账表体现「已送货 + 已付款」；采购→送货到→付款 = 一个闭环完成。对账单/采购单状态体现 已送货、已付款；闭环 = 已收货且已结清。见 4.3#9、4.4。
D要比金蝶简单：金蝶字段太细、面向大公司（含出租库等用不到的），要贴合本公司规模的轻量版。本设计刻意只保留核心流程，二/三期再扩；不做复杂质检/竞标/门户。
E先建总表、逐步迭代：先把总表建起来，字段让采购员后面一个个提、往上加（同现工作流）。一期先上"供应商+采购单+对账+汇总总表"，字段做成可扩展，按需增列。
F会提供现有采购/对账 Excel 截图作参考（表格可能不全，需重新设计）。收到截图后据此细化字段与对账表样式。

一、模块目标与范围

把现有「采购部」从"看图纸/数据表"升级为完整的采购管理子系统：管供应商、走采购全流程（请购→下单→到货→对账→付款）、出每家供应商的对账报表、并与财务部应付/付款联动，最终沉淀月/季/年采购额汇总报表。

范围内

供应商档案、采购申请/订单、到货登记、供应商对账单、应付与付款（财务联动）、采购汇总报表。

暂不含（可二期）

在线询价比价竞标、供应商门户自助、复杂质检/退换货、与外部税务系统对接。

二、角色与权限

#角色主要权限业务确认
1采购员（外协/标准件/原料/激光等分工）维护供应商、建采购申请/订单、登记到货、生成对账单；只看本人/本类数据（沿用现有按账号分工）
2采购主管审批采购订单、审核对账单、看全量采购报表
3财务应付账款、发票校验、付款审批/登记；接收对账推送
4管理层全量可见、汇总报表、审批兜底

三、采购全流程

① 请购
项目/部门提需求→ ② 选供应商
报价/比价(可选)→ ③ 采购订单
主管审批→ ④ 下单给供应商→ ⑤ 到货登记
数量/质检→ ⑥ 月度对账
供应商对账单→ ⑦ 财务核账/发票→ ⑧ 付款 完成

说明：小额/紧急采购可走简化流（采购员直接建单→主管事后审）。请购可由现有项目流程自动带出（如设计推过来的外协/原料需求）。

六、与现有系统衔接 & 分期建议

衔接点

  • 现有「采购部」按账号分工（外协/标准件/激光等）→ 直接复用为采购员分工 + 供应商分类。
  • 设计/电工推来的外协加工/不锈钢原料/激光件需求 → 转为请购/采购单来源。
  • 到货 → 复用现有「仓库」入库；财务联动 → 复用现有财务/开票的同一套财务口径。
  • 所有通知走现有「站内消息 + 企业微信」推送通道。
