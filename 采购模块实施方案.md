## 一、业务决策汇总

| # | 问题 | 决策 |
|---|---|---|
| Q1 | 采购明细数据来源 | **手工录入**，货到即录，电工清单继续走现有收件箱作参考 |
| Q2 | 到货登记与仓库入库的边界 | **完全分离**，采购模块只记账，仓库模块独立，一期互不干扰 |
| Q3 | 历史数据处理 | **录期初余额**，每家供应商录一条截止日欠款，系统从截止日后开始记新明细 |
| Q4 | 财务联动 | **加请款步骤**：采购员发起请款 → 财务审批 → 财务记录付款 |
| Q5 | 一期范围 | **不瘦身**，按需求文档 v1 全量实现一期 |

## 三、实施方案

### 开发顺序建议（先后端后前端，优先主链路）

```
第1步：数据模型
  models.py 新增 4 张表 → 确认字段无误后开始写接口

第2步：核心 CRUD 接口
  供应商 CRUD → 采购明细 CRUD → 账目一览汇总

第3步：请款流程
  PaymentRequest 接口 → 财务审批接口 → 付款回写逻辑

第4步：报表接口
  KPI/趋势/按采购员/按项目/Top供应商

第5步：前端主页面
  PurchaseMgmtView.vue 框架 → 采购明细 Tab → 账目一览 Tab → 报表 Tab

第6步：弹窗/交互
  新增弹窗 → 批量操作 → 下钻抽屉

第7步：财务侧
  FinanceView.vue 加请款审批 Tab

第8步：联调收尾
  权限验证 → 菜单/路由 → 期初余额录入功能
```

### 关键约束

- **采购明细是唯一录入入口**，账目一览和报表全部从明细+期初余额自动计算，无独立录入
- **欠款 = 收货金额合计 + 期初余额 - 已付款合计**
- **待开票 = 收货金额合计 - 已开票金额合计**
- **请款付款后回写明细**：付款金额按比例或指定分配到具体明细行的 `paid_amount`
- **供应商只能停用，不能物理删除**（有历史明细的供应商）
- **期初余额**：每家供应商录一条 `SupplierOpeningBalance`，`balance_date` = 截止日，`outstanding_amount` = 截止日欠款

## 五、核心数据模型设计

```python
class Supplier(Base):
    __tablename__ = "suppliers"
    id, name, code, category       # 分类：外协/标准件/不锈钢/激光/电气/运输…
    contact, phone, address
    tax_no, bank_name, bank_account
    settlement_type                 # 现金 / 月结 / 无账期
    credit_days                     # 月结时的账期天数
    status                          # active / inactive
    created_at

class PurchaseItem(Base):
    __tablename__ = "purchase_items"
    id
    supplier_id                     # FK → suppliers
    delivery_date                   # 送货时间
    contract_no                     # 合同编号
    project_code                    # 项目编号（字符串，不强制 FK）
    delivery_note_no                # 送货单号
    item_name, spec                 # 名称、规格型号
    qty, unit_price
    received_amount                 # 收货金额（可手改，默认=qty×unit_price）
    invoice_date, tax_rate          # 开票日期、税率
    invoice_amount                  # 开票金额
    paid_amount                     # 已付款金额（请款付款后回写）
    paid_date                       # 付款日期
    invoice_status                  # 待对账 / 已对账 / 已开票
    buyer_id                        # FK → users（录入人即采购员）
    notes
    created_at

class SupplierOpeningBalance(Base):
    __tablename__ = "supplier_opening_balances"
    id
    supplier_id                     # FK → suppliers
    balance_date                    # 期初截止日期
    outstanding_amount              # 期初欠款金额
    notes

class PaymentRequest(Base):
    __tablename__ = "payment_requests"
    id
    supplier_id                     # FK → suppliers
    requested_amount                # 请款金额
    requester_id                    # FK → users（采购员）
    status                          # pending / approved / rejected / paid
    notes
    finance_approver_id             # FK → users（财务）
    approved_at
    paid_amount, paid_date
    payment_method                  # 转账/现金…
    created_at
    # 关联明细行通过 payment_request_items 中间表
```

## 六、请款流程说明

```
采购员
  └─ 在采购明细/请款入口 → 填供应商、金额、备注 → 提交请款单
          ↓ 站内消息通知财务
财务
  └─ 财务模块"请款审批"Tab → 查看待审列表
          ├─ 审批通过 → 状态: approved → 继续走付款
          └─ 驳回 → 状态: rejected → 通知采购员
          ↓ 审批通过后
财务
  └─ 记录实际付款（付款日期/金额/方式）→ 状态: paid
          ↓ 自动回写
系统
  └─ 更新关联 PurchaseItem.paid_amount / paid_date
     重算供应商账目一览的欠款余额
```
