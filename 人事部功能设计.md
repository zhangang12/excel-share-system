# 人事部功能设计（方案稿，待确认后实施）

> 背景：用户要求"增加人事部相关的一级菜单，相关功能你设计一下"。
> 本方案按本系统一贯的「先小步可用、再逐档加深」思路设计，2026-07-11。
> 现状：系统已有 `hr` 角色（目前只能看项目目录/详单）、OA 已有部门主数据(departments)与审批流。

---

## 一、设计原则

1. **不做大而全的 HR 系统**——工资条、社保、考勤打卡都不做（录入负担会杀死使用意愿，
   和盈利改善规划里"不上工时打卡"同一判断）。只做**制造业小厂真正缺的四件事**：
   人在哪、合同/证件什么时候到期、每月人工成本多少、请假出差有没有台账。
2. **能复用就复用**：部门用 OA 的 departments；请假/出差走 OA 审批流加单据类型，人事部只做统计视图；
   提醒挂现有 overdue_scheduler；权限走现有角色 + 二级菜单权限。
3. **直接服务盈利改善规划**：规划 §五-1 明确"人工成本占总成本 25~40%，系统里一分钱都没有"，
   解法就是"每月只人工填一个各部门工资总额"。这个录入口自然归人事部——
   本模块第一期就把它做了，毛利榜"含分摊人工"口径的数据源随即就位。

---

## 二、一级菜单与权限

- 新增一级菜单 `hr`「人事部」，可见角色：`hr` + admin/manager（ROLE_MENUS 里 hr 角色追加）。
- 二级菜单(tab)全部注册进 TAB_REGISTRY（`hr:roster` 等），按账号可再收权——与全系统权限统一口径一致。
- 敏感度分层：**花名册含身份证/电话，仅 hr+管理层可见；工资总额只到部门汇总数，不碰个人工资**。

## 三、页签设计（第一期 = ①+③，第二期 = ②，第三期 = ④）

### ① 👥 员工花名册（第一期）

新表 `employees`（与登录账号弱关联，可挂可不挂）：

| 字段 | 说明 |
|---|---|
| name / id_card / phone / emergency_contact | 基本信息（身份证选填） |
| department_id | 复用 OA departments |
| position | 岗位（自由文本） |
| hire_date / regular_date | 入职日期 / 转正日期 |
| contract_end | **合同到期日**（提醒的核心字段） |
| status | 试用 / 在职 / 离职（+离职日期 leave_date） |
| user_id | 可选关联系统账号（离职时提示管理员停用账号） |
| note | 备注 |

- 列表：按部门/状态筛选、搜索；新增/编辑/离职登记；**Excel 导入**（模板同供应商导入的做法）。
- 顶部 KPI：在职人数 / 试用期人数 / 30 天内合同到期数 / 本月入职·离职。
- **自动提醒**（挂 overdue_scheduler，每日扫，7 天窗口去重）：
  - 合同到期前 30 天 → hr + 管理层；
  - 试用期满（转正日期）前 7 天 → hr + 部门主管；
  - 关联账号的员工离职登记时 → 提醒 admin 停用该登录账号。

### ② 📜 证件台账（第二期）

新表 `employee_certs`：员工 × 证件（电工证/焊工证/叉车证/特种作业证…自由类型 + 到期日 + 附件照片）。
到期前 60/30 天提醒 hr + 员工所在部门主管。制造业年检场景刚需，但可以等花名册跑顺再上。

### ③ 💰 部门月度工资总额（第一期，盈利改善的关键拼图）

新表 `payroll_monthly`：`(month, department_id, total_amount, note)`，一月一部门一行。

- 页面：选月份 → 每部门一格填总额 → 合计自动算；只允许 hr/管理层录入与查看。
- 下游（第二步，另行实施）：毛利榜/支出总览增加"含分摊人工"口径——
  按当月各部门活跃项目的任务时间戳权重分摊（规划 §五-1 的低成本解法）。
  **先把数录起来，口径打通是纯只读计算，随时能加。**

### ④ 🏖 请假/出差台账（第三期）

- OA 增加「请假」单据类型（doc_type=leave：类型/起止/天数/事由），走现有 OA 审批流（部门主管→hr 抄送）。
- 人事部 tab 只做**统计视图**：按人/按部门/按月的请假天数汇总（纯只读，无新审批逻辑）。

## 四、实施顺序与工作量

| 期 | 内容 | 预估 |
|---|---|---|
| 第一期 | 一级菜单 + 花名册(含Excel导入+三类提醒) + 部门月度工资总额 | 一次会话可交付 |
| 第二期 | 证件台账 + 到期提醒 | 小半次会话 |
| 第三期 | OA 请假单据类型 + 人事统计视图；毛利榜"含分摊人工"口径 | 分别半次会话 |

## 五、待用户确认的问题

1. 第一期范围（花名册 + 工资总额）这样切可以吗？
2. 身份证号要不要存？（存的话仅 hr+管理层可见；不存也不影响其他功能）
3. 工资是"部门月度总额"一个数（推荐，够毛利分摊用、不碰个人隐私），还是要到个人？
4. hr 角色目前是谁在用？要不要顺便把 hr 账号的项目目录/详单权限收掉（现在能看全部项目）？
