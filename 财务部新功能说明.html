💼 财务部新功能说明

同辉项目管理系统 · 2026-07-11 · 适用对象：财务部 / 管理层
一句话总览：财务部这次新增了 4 个页签，回答四个一直答不上来的问题—— 钱花哪了（支出总览）、哪个项目赚钱哪个亏钱（项目毛利）、 成本数字可不可信（成本审计）、现金会不会断（资金面板）。 全部基于系统里已有的数据自动计算，不需要任何额外录入。
💸 支出总览 📈 项目毛利 🕳️ 成本审计 💰 资金面板 🏷️ 账期到期标签 🔔 自动提醒 🔐 权限与准备

💸 支出总览

财务部 → 支出总览 全公司的钱花在哪，按月一张表看全
看什么选定年份的 4 个合计（年总支出 / 采购付款 / 安装售后费用 / OA 业务报销）+ 逐月明细表。
口径采购付款按付款日期归月 + 安装/售后费用（售后部已审批的）+ OA 业务/报销费用（已审批，财务核定金额优先于申请金额）。
防重复OA 里的「采购申请」大类不计（钱已经在采购付款里）；仓库材料领用也不重复计（钱在采购付款里，领用是项目成本口径，见项目毛利）。
表格下方如出现黄色提示「另有 ¥X 已付款但未记付款日期」——说明有采购付款没填付款日期，进不了月份表。到采购明细补上付款日期即可。

📈 项目毛利（红黑榜）

财务部 → 项目毛利 哪个项目赚钱、哪个项目亏钱，一屏排出来
毛利 = 合同额 − 材料领料成本 − 直发/外协采购 − 安装/售后费用
成本腿怎么算
材料领料成本仓库出库到该项目的数量 × 物料入库加权平均价
直发/外协采购没经过仓库过账的采购收货金额，按订单编号挂到项目（外协加工费、直发件都在这条腿）
安装/售后费用售后部已审批的安装/售后费用

▸ 同一笔钱只走一条腿：经仓库收货的采购走「领料」腿，不会和「直发采购」重复计算。

怎么用

  • 默认亏得最多的排最前（红榜在上），亏损行整行淡红；点列头可按合同额/成本/毛利/毛利率重新排序。
  • 左上角下拉可切换 按客户 / 客户类型 / 销售员 / 订单类型 / 机型 / 年份 汇总——比如按机型看哪类设备总是亏。
  • 每行带「成本完整度」标签：完整 = 数字可信； 领料缺价 N 项 = 有物料没入库价格，成本被低估（去成本审计页清理）； 无销售台账 / 合同额为 0 = 收入侧缺数。
  • 表格下方「售后侵蚀 Top5」：售后费 ÷ 合同额 排名（可切机型/客户），定位返修高发的设备和客户。
⚠️ 口径必读：这是「材料边际贡献」，不含人工工资、运费等系统外成本。 排名用于项目之间比较非常可靠，但绝对值 ≠ 净利润——费工不费料的深度定制项目，实际盈利会比榜上显示的差。

🕳️ 成本审计

财务部 → 成本审计 毛利榜可信度的前提：三张"漏钱清单"不清零，毛利就系统性虚高

页面顶部常驻两个数：本月未归集到项目的成本 和 累计未归集。目标是把它们清到零。

清单是什么怎么修
🕳 无主领料 出库时没挂项目的领料——这部分材料钱不进任何项目成本，在全系统"蒸发" 每行右侧直接补选项目 → 点「归集」，成本立即回到该项目毛利
💸 无价入库 仓库收货时没填价格的流水——压低库存金额和项目材料成本 标「可回填」的（采购后来补了价）点顶部「⚡ 一键回填」；标「采购也没价」的先去采购明细补单价（现在补价会自动同步到收货流水）
👻 孤儿采购 订单编号既不是项目编号、也不在字典里的采购——成本挂空 到采购明细把订单编号改对（下单入口已改为只能下拉选择，新的不会再产生）
⚖️ 双口径对账 逐项目对比「采购收货金额」vs「仓库领料成本」，差异倒序 差异大 = 漏归集重灾区：材料还压在库存没领，或者领了没挂项目

配套的三个"堵漏"机制（自动生效）

  • 仓库出库必须选项目；确属非项目领用（车间耗材等）要勾选「非项目领用」并填原因。
  • 采购补价自动回写收货流水——不再产生新的无价入库。
  • 冲红口径修正：被冲红的单据不再计入加权价和项目成本。

💰 资金面板

财务部 → 资金面板 账面有利润、银行没有钱？四个现金盲区一屏看清

顶部 4 个数：逾期应收 / 预付敞口 / 逾期应付 / 呆滞库存。下设五个子页：

⏰ 逾期应收 —— 该找谁催钱

  • 尾款按约定日、发货款按发货日计龄，30 / 60 / 90 / 90+ 天分档；按客户和销售各排 Top10。
  • 系统会自动每周催办逾期尾款并逐级升级（见下方「自动提醒」）。

💳 预付敞口 —— 押在供应商那里的钱

  • 已付款但货未到的采购，按供应商列出押款金额和已押天数（超 30 天标红）——照着催交货。

📆 应付账期 —— 有账期别提前付，逾期未付防断供

  • 到期日 = 到货日期 + 供应商账期天数。左边「逾期未付」（断供风险），右边「14 天内到期」（排程付款）。
  • 「历史提前付」统计：有账期却提前付掉的金额 + 平均白白放弃了多少天免息——这是纯送给供应商的资金占用。
  • 下方黄条提示哪些供应商没维护账期天数（算不了到期日），请在供应商档案补上。

🧊 呆滞库存 —— 被锁死的现金

  • 超过 90 / 180 / 365 天没有出库动销的存货 × 加权均价，从大到小排；「从未出库」单独标红。
  • 每行带采购回溯：这批货当初是谁、为哪个项目买的。
  • 附「安全库存体检」：安全库存定得远超月均用量的（压钱型）、现存常年低于安全线的（常报警型）。

📉 13 周现金排程 —— 未来三个月的钱够不够

  • 预计流入 = 有约定日期的尾款按周排（已逾期的记在「已到期/随时」行）； 预计流出 = 已批未付请款 + 应付到期 + OA 待付款（已进请款单的应付不重复计）。
  • 累计净额为负的周整行标红并打「缺口」标签——提前看到哪一周可能断，要么提前催收，要么把付款往后排。
  • 表下另列：没有约定日期的已发货应收（催回即是流入）。
⚠️ 局限：系统目前没有「实际到账流水」登记（属规划第三档），流入侧是按合同约定日的应收，不是实收。约定日期不填、流入就排不进来。

🏷️ 请款审批 / 付款页的「账期到期」标签

财务部 → 请款审批 · 付款 按到期日排程付款，不用去翻供应商档案心算

每张请款单现在多一列「账期到期」（= 明细最早到货日 + 该供应商账期天数）：

标签含义建议动作
逾期 N 天已过应付到期日优先付，防断供
距到期 ≤ 7 天 / 今天到期快到了安排进本周付款
距到期 N 天（>7）还有账期不必提前付——免息资金先留在手里
未维护账期供应商档案没填账期天数补档案后自动出现倒计时

🔔 两个自动提醒（不用人盯）

系统后台自动扫描，推送到站内消息 / 企业微信
提醒触发规则推给谁
尾款逾期每周升级催办 尾款过了约定日仍未收：每 7 天催一次 逾期第 1 周 → 销售本人；第 2 周 → 加抄销售主管；第 3 周起 → 加抄管理层，收回为止
（到期前 14 天的一次性提前提醒保持不变）
售后侵蚀预警 售后费用审批通过时，该项目累计售后费 ÷ 合同额 > 5% 管理层（提示关注该项目/机型的质量成本）

🔐 谁能看 & 用前准备

权限

  • 四个新页签对 财务 / 财务主管 角色开放，管理层（admin/manager）全部可见。
  • 管理员可在「用户管理 → 二级菜单权限」按账号精细控制某人能看哪几个页签。

数字要准，四件事要做到（数据质量清单）

  1. 供应商档案补「账期天数」——应付到期、账期标签、13 周排程都靠它（一次性补齐即可）。
  2. 采购付款要填付款日期——不填进不了支出总览月份表。
  3. 出库必须挂项目（系统已强制）——历史无主领料在成本审计页尽快归集。
  4. 销售台账的尾款约定日期要填——不填就没有账龄、没有催办、进不了现金排程。
建议节奏：每周一打开「资金面板」过一遍逾期应收和 14 天内到期应付；每月初看「项目毛利」和「支出总览」；「成本审计」清单清到零后，毛利榜数字才完全可信。
