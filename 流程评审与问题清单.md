# 多部门协作流程 · 评审与问题清单

> 评审对象：`demo/增量UI设计图.html`（统一原型）+《需求分析》《业务确认结论》《仓库管理功能设计》。
> 目的：把整条链（销售→设计/电工/生产→钣金/采购/仓库→物流）串起来前，先暴露衔接漏洞、逻辑空洞与待决策点。
> 标注：严重度 **P0 阻断 / P1 重要 / P2 优化**；类型 **【决策】需业务拍板 / 【可改】原型可直接闭环 / 【开发】落地注意**。

## 一、端到端流程串接总图（目标态）

```
销售下单（建项目 + 收货信息E2 + 派部门）
  │
  ├─[设计部负责人分派→设计师]  设计：①四表校验→完成 ②说明书/铭牌→物流 ③PDF图纸包→钣金组
  ├─[电工部负责人分派→电工]    电工：接单→采购清单→采购；完成必传电路图→物流
  └─[生产主管下单→装配组]      生产：完成→通知物流（无产物，仅状态信号）
        │
        ├─ 仓库：项目领料出库(挂项目) / 发货清单→物流；库存只读给设计
        ▼
  物流发货部  ──【闸门：设计&电工&生产 均完成 D5】──> 汇总资料+收货信息 → 上传发货单 → 已发货
        ▼
  项目「已完成」（联动项目目录状态）
横切：企微通知 / 逾期→对应部门主管 · 月度报表 · 导出审批 · 角色可见性A3
```

## 三、P0 阻断项（详述）

### P-01 「钣金装配」归属冲突 【决策】
- 现状：需求说"项目详情的**钣金装配 Excel 挪到钣金组**"；同时设计完成要校验"四个 Excel 表"含**钣金装配**（D2）。
- 风险：挪走后四表只剩三表，设计完成校验口径崩。
- 建议（择一）：**A** 钣金装配仍留四表参与设计校验，钣金组只是"引用/可见"同一张表；**B** 设计校验改为"三表+钣金图纸包"。倾向 A（最小改动、不破坏现有四表）。

### P-02 部门先后依赖未建模 【决策+开发】
- 现状：销售可同时/任意给设计、电工下单；生产主管独立下单；彼此无先后约束。
- 风险：电工接线依赖设计图、生产装配依赖设计+采购到料；无依赖会出现"电工先于设计完成""无图就装配"。
- 建议：明确依赖与是否**强约束**：① 软提示（可并行，仅看板提醒）② 硬门禁（上游未完成不可开始下游）。倾向**软提示**（制造业常并行赶工），物流处做硬门禁（P-03）。

### P-03 发货前置未强制（D5）【可改 ✓ 本轮已串接】
- 现状：物流「已发货」随时可点。
- 修复：发货按钮加**闸门**——该项目设计/电工/生产的已下单任务全部 `完成` 才可发，否则禁用并显示缺口。

### P-04 项目「已完成」判定缺失 【决策】
- 现状：项目状态（进行中/已完成/已归档）无人联动；各部门任务 done 后项目仍"进行中"。
- 建议：定义项目完成 = **物流已发货**？还是 **三部门完成**？归档由谁触发？需口径。倾向：三部门完成→"待发货"，物流已发货→项目"已完成"。

### P-05 部门页面可见性未按 A3 【可改 ✓ 本轮已串接】
- 现状：所有角色都能看到全部菜单。
- 修复：菜单按角色过滤（本部门成员 + 管理层）；管理层/超管见全部；设计师额外获「仓库（只读）」入口。

### P-06 完成可逆的数据一致性 【决策+开发】
- 现状：完成→已推说明书/电路图给物流、采购清单给采购；改回进行中后，下游已收资料不回收。
- 风险：物流/采购看到"幽灵资料"，发货闸门误判。
- 建议：改回进行中时**自动冲销/撤回**对应下游推送（本轮原型已做轻量撤回）；真实系统需事务+审计。

## 四、P1 重要项（要点）

- **P-07 生产完成信号**【可改✓】：物流"生产部"列由"找资料"改为显示**生产状态**（未下单/进行中/已完成）。
- **P-08 仓库清单进物流**【可改✓】：物流看板新增「仓库发货清单」列，承接仓库组上传。
- **P-09 下单触发**【决策】：电工、生产是否都由**销售**一次派？还是设计完成后再派电工/生产？建议销售可**勾选多个部门**一次下单（设计/电工），生产仍由生产主管按节点下单。
- **P-10 通知人范围**【可改✓】：设计完成通知人从"全公司"收窄为**相关角色下的人员**（D3）。
- **P-11 逾期精准**【可改✓】：逾期推**对应部门主管**（设计→设计负责人，电工→电工负责人，生产→生产主管），不再群发。
- **P-12 设计产物入口**【优化】：建议把"PDF图纸包→钣金"也并入设计**完成弹窗**统一上传，减少多处操作。
- **P-13 作废重下闭环**【决策】：作废=本项目编号该部门任务作废；管理层重下是**新任务**（原任务留痕作废），原已传资料不带入。需确认。
- **P-14 设计师看库存**【可改✓】：随 A3，给设计师"仓库（只读）"菜单。
- **P-15 导出免审**【决策】：G1"全部导出都要审批"——管理层导出是否也要走审批？建议管理层免审、其余必审。
- **P-16 四表判定**【开发】："导入过 Excel"需在数据表上加"是否有导入记录"标记，仅手工加行不算。

## 五、P2 优化项（要点）
- **P-17** 报表范围是否含销售/钣金/仓库/物流（建议至少纳入销售下单量、仓库出入库量）。
- **P-18** 完成效率：预计用时 0 按 1 天（C3）；当天完成算按时（C2）—原型口径对齐。
- **P-19** 制图列隐藏：配置持久化（全局/按角色？）+ 存量制图值由设计部模块承接展示。
- **P-20** 企微：先做"用户↔userid"绑定管理页，凭证到位即通。
- **P-21 / P-23** 项目与收货信息多入口：统一编号唯一校验、收货信息单一权威来源（销售录入为准）。
- **P-22** 采购合并：迁移脚本把标准件/外协采购员并入采购部并迁字段权限（幂等）。

## 六、本轮已在原型中「串接/修复」的项
1. **发货闸门（P-03）**：物流「已发货」需设计/电工/生产已下单任务全部完成，未满足则禁用并提示缺口。
2. **物流看板重构（P-07/P-08）**：列改为 设计资料｜电工资料｜**生产状态**｜**仓库发货清单**｜收货信息｜发货闸门，全链资料/状态一屏可见。
3. **角色可见性（P-05/P-14）**：侧栏菜单按角色过滤；设计师获"仓库（只读）"入口；管理层见全部。
4. **逾期精准推送（P-11）**：逾期改推对应部门主管。
5. **完成可逆撤回（P-06 轻量）**：改回进行中时，撤回该任务已推送给物流/采购的资料。
6. **通知人收窄（P-10）**：设计完成通知人限相关角色（不再全公司）。

> 其余【决策】项见第三/四章，需业务拍板后再落地；【开发】项在进入开发时按《业务确认结论》执行。
