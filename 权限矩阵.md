# 权限矩阵（增量UI原型现状汇总）

> 对应 `demo/增量UI设计图.html`。✓=可见/可操作，❌=不可见，「仅本人」=行级数据隔离（myProjects）。
> 管理层 / 超级管理员 = 全部权限（下表不再逐列标，默认全 ✓）。

## 一、项目目录 / 项目详单（数据范围）

| 角色 | 项目目录 | 项目详单 |
|---|---|---|
| 管理层 | 全部 | 全部 |
| 销售员 | 仅本人 | ❌（无此菜单） |
| 销售主管 | 全部 | ❌ |
| 设计师 | 仅本人 | 仅本人 |
| 设计部负责人 | 全部 | 全部 |
| 电工 | 仅本人 | ❌ |
| 电工部负责人 | 全部 | ❌ |
| 生产员（装配组） | 仅本人 | 仅本人 |
| 生产部主管 | 全部 | 全部 |
| 钣金组 | 全部 | 全部 |
| 采购部 | 全部 | 全部 |
| 仓库员 / 仓库主管 | 全部 | 全部 |
| 物流发货部 | 全部 | 全部 |

> 「仅本人」判定：销售员=自己下单的(salesUid)；设计师=自己是设计师或被分派设计任务；电工/装配组=被分派的本部门任务。

## 三、关键功能权限

| 功能 | 可操作角色 |
|---|---|
| 销售下单（建项目+台账+派部门） | 销售员 / 销售主管 / 管理层 |
| 销售台账**编辑** | 销售主管 / 管理层 |
| 销售台账**合计** + 按销售员筛选 | 销售主管 / 管理层 |
| 上传合同 / 开票申请 | 销售员 / 销售主管 / 管理层（看自己范围） |
| 项目目录「＋下单」浮动按钮 | 管理层（派设计/电工）· 生产部主管（派生产） |
| 部门负责人**分派** | 设计部负责人（设计）· 电工部负责人（电工） |
| **部门报表**（仅本部门数据） | 设计/电工/生产 部门负责人各自 + 管理层 |
| **月度工作报表**（全公司） | **仅管理层** |
| 仓库出入库 / 物料 / 收发存 / 发货清单 | 仓库员 / 仓库主管（写）· 管理层（读）· 设计师（只读查库存） |
| 设计完成「四表校验」+ 上传产物 | 设计师（本人任务） |
| 物流确认发货（回写发货日期） | 物流发货部 |
| 导出（项目/报表/库存等） | 管理层**免审**；其余角色→申请→管理层审批 |
| 完成可逆 / 冲红 / 列隐藏(制图) | 见各模块（制图列对业务无感知=逻辑删除） |
| 消息推送人（接收逾期/预警推送） | 销售主管 / 各部门主管 / 仓库主管（在权限管理标记） |
